# Link & Tag Parity 审计报告

- 参考站：`https://wangyu.me`
- 审计范围：`/`、`/list/`、`/categories/`、`/tags/`、`/about/`、文章页（本地：`/posts/sample-post-001/`；参考：`/posts/ml/flash-decoding/`）
- 说明：仅做分析，不修改项目源码。

## 链接一致性（通过项/问题项）

### 通过项
- 公共导航结构与参考站一致：`首页 / 列表 / 分类 / 标签 / 关于`，顺序一致，且在 `/list/`、`/categories/`、`/tags/`、`/about/` 的 active 状态正确。
- `/list/`、`/categories/`、`/tags/` 页中的文章链接层级正确（均指向 `../posts/.../`），抽样与脚本检查未发现缺失目标。
- `/categories/` 与 `/tags/` 的 chip 锚点与分组标题 `id` 一一对应（无悬空锚点）。
- 首页存在“查看全部”入口，并能正确跳转到列表页。

### 问题项
1. **首页导航“首页”链接写法不一致**  
   - 本站：`href=""`（空链接）
   - 参考：`href="/"`
   - 影响：功能通常可用，但链接语义与层级表达不统一；在带 query/hash 场景下行为可能与预期根路径不完全一致。

2. **文章页缺少 taxonomy 回链（分类/标签）**  
   - 本站文章页（`/posts/sample-post-001/`）中“发布时间｜分类”是纯文本，无可点击分类/标签链接。
   - 参考文章页存在 `分类 -> /categories/#...`、`标签 -> /tags/#...` 的回链。
   - 影响：文章页到分类/标签页的结构链路不闭环（层级导航缺一跳）。

3. **`/about/` 外链与参考站非一一对应**（若目标是严格复刻）  
   - 本站为占位符：`https://example.com`、`mailto:hi@example.com`
   - 参考站为真实 GitHub/邮箱链接。
   - 影响：若审计目标是“超链接一一对应”，则属于错链/缺失对应链接；若出于脱敏可视为有意差异。

## 标签格式一致性（通过项/问题项）

### 通过项
- `tags` / `categories` 的 chip 视觉规则与参考站一致（CSS 对比）：
  - 字号：`font-size: 12px`
  - 边框：`1px solid #ddd`
  - 内边距：`padding: 2px 10px`
  - 间距：`margin-right: 5px; margin-bottom: 5px`
  - hover/active：`color: #fff; background-color: #000`
- 计数表现一致：均为“名称 + 空格 + `(数字)`”的内联文本形式（如 `devops (18)`）。
- chip 与分组标题锚点匹配正常（格式与交互路径一致）。

### 问题项
- 本次审计在“字号、chip 边框、间距、hover、计数表现”5个维度上，**未发现与参考站的实质样式偏差**。

## 修复建议（按优先级）

1. **P0：补齐文章页 taxonomy 回链**  
   在文章元信息区增加可点击的分类/标签链接（至少补 `分类 -> /categories/#...`，建议同时补 `标签 -> /tags/#...`），保证内容页与聚合页双向可达。

2. **P1：统一首页“首页”导航链接为显式根路径**  
   将首页导航的 `href=""` 统一为 `href="/"`（或与站点其余页面一致的根路径写法），消除层级表达歧义。

3. **P2：明确 `/about/` 外链策略**  
   - 若目标是严格 1:1 复刻：替换占位符为参考站对应链接；
   - 若目标是脱敏发布：保留占位符，但建议在文档中注明“有意与参考站不同”，避免后续审计反复报错。
